Face recognition lock system

Esp-32 cam, Arduino IDE, Relay module

Face recognition lock system

Description

Introduction

In an era where security is paramount, I embarked on a project to create a Face Recognition Doorlock System using readily available components such as ESP32-CAM, a relay module, and a solenoid lock. This innovative solution not only enhances security but is also cost-effective, power-efficient, and ensures data privacy by storing information locally on an SD card.

Components Used

ESP32-CAM :
A versatile and cost-effective microcontroller with built-in Wi-Fi and camera capabilities.

Relay Module :
To control the solenoid lock, providing a secure means of unlocking the door.

Solenoid Lock :
A reliable and efficient lock mechanism that responds to the relay module's signal.

Face Recognition Algorithm :
Utilizing the ESP32-CAM's capabilities, I implemented a face recognition algorithm. Upon activation, the system captures an image and compares it with faces enrolled locally on the device. A successful match triggers the relay module to send a signal, unlocking the door.

Local Data Storage :
Unlike many commercial solutions, this system prioritizes user privacy by storing face data locally on an SD card. This approach not only simplifies the architecture but also ensures faster processing times.

Arduino IDE Code Implementation :
The project was developed using the Arduino IDE, offering a user-friendly interface for coding and uploading. The code captures images, processes them locally, and communicates with the relay module to control the solenoid lock. The simplicity and accessibility of Arduino IDE make it an ideal choice for DIY enthusiasts.

Advantages

Cost-Effective :
The components used are affordable and readily available, making the system accessible to a wide audience.

Power-Efficient :
The ESP32-CAM's low power consumption contributes to a sustainable and energy-efficient solution.

Data Privacy :
By avoiding cloud storage, the system protects user data from potential security breaches.
